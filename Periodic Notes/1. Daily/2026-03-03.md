## PREPAY-780

@KbankPreparationWebControllerV2 클래스에 케이뱅크로부터 Callback API 호출을 받는 Controller 프로세스 개발을 시작해보자. (참고: @CustomerCertificationWebControllerV1 - Callback 요청받아 처리 후 returnWebUrl redirect 처리 프로세스 참고 필요)

@KbankPreparationWebControllerV2 에서 요청을 받아 처리할 프로세스는 다음과 같아.

1. k-bank 상품 이용 완료 callback 요청받는다.
2. `request.customerId(customDi)` 를 조건으로 `KbankCustomerMaster` DB 조회
3. `KbankCustomerMaster.hashCi` 와 `request.type` 를 조건으로 `KbankPreparationCache` Cache 조회
4. `request.clientWebviewCode` 와 `KbankPreparationCache.clientWebviewCode` 일치 여부 확인
	1. 일치하는 경우, 후속 프로세스 진행
		1. `reuqest.type == ACCOUNT` 인 경우
			1. `KbankOutboundApi.kbankAccounts` 인터페이스 호출하여 제휴 계좌 목록 1건 조회
			2. `KbankCustomerSavedEventPayload.processStatus: "AFFILIATED_COMPLETED"` 로 하여 `KbankCustomerSavedEvent` 이벤트 발행
			3. `MpsAccountSessionClient.getAccountSession` 인터페이스 호출하여 `sessionUrl` 조회
			4. 전달받은 `sessionUrl` 를 최종 API redirect 응답 처리
		2. `request.type == CARD or PROMOTION` 인 경우
			1. 후속 프로세스 없이 종료 처리
	2. 일치하지 않는 경우, 예외 발생 처리

#### k-bank 상품 이용 완료 callback API 정보
- URL: `/v2/preparation/kbank/{type}/result`
```json
{
	"customerId": "<customDi>",
	"type": "<KbankPreparationType>",
	"clientWebviewCode": "<clientWebviewCode>"
}
```

#### MpsAccountSessionClient.getAccountSession 요청 정보
```json
{
  "accessToken": "<accessToken>",
  "sessionType": "REGISTER_PARTNER_PAYMETHOD",
  "payKind": "TRANSFER",
  "partnerAccountNumberEnc": "<KbankOutboundApi.kbankAccounts 결과 계좌 정보 KMS 암호화값>",
  "partnerBankCode": "BK89"
}
```
