
## 머니 포인트 지급 시스템 디자인
- 무신사 5만원 상품권 지급
- 1만원 이상 머니 사용하면 5천원씩 페이백 포인트 지급

### 머니 구분
- 유상 머니
	- 은행 계좌와 동일한 구조의 원장
	- 회원 기준 충전/사용 거래 원장
- 상품권
	- 회원 + 바우처번호 기준 지급/ 사용 거래 원장
	- 인출 가능 여부를 통해 현금 인충 가능 여부 구분
	- 유상(구매) 
	- 무상(포인트)
- 포인트
	- 상품권과 동일한 구조의 거래 원장
	- 만료 일자가 있어서 포인트 만료 처리
- 캠페인
	- 상품권 발급/사용 여부 관리를 위한 정보 관리 원장

### 새해맞이 이벤트 상품권 대량 지급 기능 구축

- 26/01/16 금요일 시스템 디자인 리뷰
- 26/02/09 실제 포인트 지급 예정
#### 기능 요구사항
1. 식별된 대상자 머니 포인트 지급
2. 포인트 지급 사유 기록 필요(건별로)
3. 포인트 지급 실패 건 재처리 기능 필요
#### 비기능 요구사항
1. 대용량 처리(100만건+/1M+)
2. 고성능 처리
3. 기능 재활용 가능한 아키텍처 요구
	1. 캠페인별로 대용량 포인트 지급 필요

- 실제 포인트 지급은 머니 시스템으로 API 호출하여 처리
