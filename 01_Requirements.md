
## Intro

### 대량 포인트 지급 시스템
- 주문사(무신사)에서 페이먼츠의 머니 사용하여 결제한 회원들을 대상으로 포인트를 지급하기 위한 대량 포인트 지급 시스템 구축
- 최대 지급 대상 회원 수는 100만(1M) 까지 지급 필요
- 포인트 지급 사유를 지급 거래 row 별 기록 필요
- 포인트 지급 실패 거래 건들은 별도 재처리 프로세스를 통해 지급 처리 필요
- 해당 대량 포인트 지급 시스템은 캠페인(포인트 지급 이벤트 등) 기준으로 재활용이 가능한 데이터 모델링 필요

#### 기능적 요구 사항
- 식별된 포인트 지급 대상자 목록 기준 머니 포인트 지급
- 포인트 지급 row 별 지급 사유(reason) 저장
- 포인트 지급 실패 건 재처리 프로세스 구축

#### 비기능적 요구 사항
- 1M(100만)+ 대용량 row 데이터 처리 필요
- 고성능 처리 필요
- 캠페인별 대용량 포인트 지급 가능한 데이터 모델링 및 시스템 아키텍처 구현 필요

##### 설계 제외 사항
- 대용량 회원 데이터 수집에 대한 부분 제외
	- 이미 수집된 대용량 회원 데이터는 DB 에 저장된 상황
- 실제 포인트 지급은 `money(머니)` 시스템을 통하여 거래 발생
	- `money(머니)` 시스템의 지급 API 호출 추상화하여 설계 반영
